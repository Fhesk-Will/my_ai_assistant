## 概述

L1 是系统对外的唯一入口，负责：
- 启动 FastAPI 应用并注册所有路由
- 处理 CORS、请求指标采集
- 托管 Vue 3 前端静态文件
- 将 HTTP 请求转发给 `CognitionEngine`

---

## 应用工厂 `src/l1_ui/server.py`

```python
def create_app(config: AppConfig) -> FastAPI:
    app = FastAPI()
    app.add_middleware(CORSMiddleware, allow_origins=config.cors_origins, ...)
    app.add_middleware(MetricsMiddleware)

    # 初始化核心组件
    memory_manager = MemoryManager(config.db_path)
    memory_system  = MemorySystem(config)
    engine         = CognitionEngine(config, memory_manager, memory_system)

    # 注册路由
    init_chat_routes(app, engine)
    init_memory_routes(app, memory_system)
    init_models_routes(app, config)
    init_observability_routes(app)

    # 静态文件：优先 frontend/dist，回退到 frontend/
    app.mount("/", StaticFiles(directory=..., html=True))
    return app
```

`create_app()` 是唯一的组件装配点。所有依赖（`MemoryManager`、`MemorySystem`、`CognitionEngine`）在此处实例化，通过 `init_*_routes()` 注入到各路由模块的模块级变量中。

## 路由

### Chat 路由 `routes/chat.py`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/chat` | 发送消息，返回 AI 回复 |
| `GET` | `/api/history` | 获取当前 session 的聊天记录（`?limit=20`） |
| `GET` | `/api/sessions` | 列出所有 session |
| `POST` | `/api/sessions/rename` | 重命名 session |
| `DELETE` | `/api/sessions/{session_id}` | 删除 session 及其消息 |

**请求/响应结构：**

```python
# POST /api/chat
class ChatRequest(BaseModel):
    message: str
    session_id: str

class ChatResponse(BaseModel):
    reply: str
    session_id: str
```

`chat()` 路由直接调用 `engine.chat(message, session_id)`，用 `try/except` 包裹，异常时返回 HTTP 500。

---

### Memory 路由 `routes/memory.py`

所有端点均为只读查询，直接调用 `MemorySystem` 的各子管理器：

| 路径 | 数据来源 |
|------|---------|
| `/api/memory/profile` | `UserProfileManager.load()` |
| `/api/memory/persona` | `PersonaManager.load()` |
| `/api/memory/episodic/summaries` | `EpisodicMemoryManager.get_recent_summaries()` |
| `/api/memory/episodic/tasks` | `EpisodicMemoryManager.get_active_tasks()` |
| `/api/memory/episodic/events` | `EpisodicMemoryManager.get_events()` |
| `/api/memory/skills` | `SkillRegistryManager.get_all_skills()` |
| `/api/memory/status` | `MemorySystem.get_memory_status()` |
| `POST /api/memory/analyze` | `MemorySystem.process_conversation()`（手动触发分析） |

### Models 路由 `routes/models.py`

| 路径 | 状态 |
|------|------|
| `GET /api/models` | 返回 `models.yaml` 中定义的模型列表（已实现） |
| `POST /api/models/switch` | 切换当前模型（TODO，尚未实现） |

### Observability 路由 `routes/observability.py`

| 路径 | 说明 |
|------|------|
| `GET /api/observability/metrics` | 最近 N 条请求指标（`?limit=100`） |
| `GET /api/observability/summary` | 聚合统计（总请求数、平均耗时、各状态码分布） |

数据来源是 `MetricsMiddleware` 的内存缓冲区，不持久化。

## 中间件 `middleware/metrics.py`

```python
class MetricsMiddleware(BaseHTTPMiddleware):
    _metrics: list[dict] = []   # 类变量，最多保留 1000 条

    async def dispatch(self, request, call_next):
        start = time.time()
        response = await call_next(request)
        duration_ms = (time.time() - start) * 1000
        self._metrics.append({
            "path": request.url.path,
            "method": request.method,
            "status_code": response.status_code,
            "duration_ms": duration_ms,
            "timestamp": datetime.utcnow().isoformat()
        })
        if len(self._metrics) > 1000:
            self._metrics.pop(0)
        return response
```

`get_recent_metrics(limit)` 和 `get_metrics_summary()` 是模块级函数，供 observability 路由调用。

## 端口冲突处理

`server.py`（根目录入口）在启动前调用 `check_port(host, port)` 检测端口占用，在 Windows 上通过 `socket.connect_ex()` 实现，端口被占用时打印警告但不强制退出（由 uvicorn 自行报错）。

## 静态文件服务策略

```python
# 优先使用构建产物
if os.path.exists("frontend/dist"):
    static_dir = "frontend/dist"
else:
    static_dir = "frontend"

app.mount("/", StaticFiles(directory=static_dir, html=True), name="static")
```

`html=True` 使所有未匹配路由回退到 `index.html`，支持 Vue Router 的 history 模式。
