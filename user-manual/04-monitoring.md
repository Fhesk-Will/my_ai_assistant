## 打开监控面板

点击侧边栏底部的「**监控面板**」链接，即可跳转到监控页面。

监控面板是一个独立页面，不影响正在进行的对话。

---

## 返回对话

在监控面板顶部，点击「**← 返回对话**」链接，即可回到对话界面。

## 概览指标（顶部四个卡片）

### 总请求数

自服务启动以来，助手处理的 API 请求总次数。每发送一条消息、加载一次历史记录，都会计入此数字。

### 平均响应时间

所有请求的平均处理耗时，单位为毫秒（ms）。

- 正常范围：通常在几百毫秒到几秒之间，取决于网络状况和 LLM 响应速度
- 数字偏高：可能是网络延迟或 LLM 服务繁忙，属正常波动

### 近期错误

最近出现的请求错误次数（HTTP 状态码 400 及以上）。

- 显示 `0`：运行正常
- 显示非零数字：可查看下方「近期请求记录」定位具体错误

### 记忆状态

显示记忆系统是否正常运行：

| 显示 | 含义 |
|------|------|
| **正常** | 记忆系统已加载，可正常读写 |
| **离线** | 记忆系统未能初始化，长期记忆功能不可用 |

---

## 记忆系统详情（中间区域）

展示助手当前积累的记忆数量，共五个模块：

### 用户画像

| 字段 | 含义 |
|------|------|
| 兴趣数量 | 助手已记录的你的兴趣领域数量 |
| 技能数量 | 助手已记录的你的技术技能数量 |

这些信息来自你在对话中自然提及的内容，助手会自动提取并保存。

### 人格版本

| 字段 | 含义 |
|------|------|
| 人格名称 | 当前 AI 助手的名字（默认：灵曦） |
| 版本号 | 人格配置的当前版本，每次调整后递增 |

### 情景记忆

| 字段 | 含义 |
|------|------|
| 对话摘要数 | 已分析并保存摘要的历史对话数量 |
| 活跃任务数 | 当前处于「进行中」或「待办」状态的任务数量 |
| 关键事件数 | 已记录的重要事件数量 |

### 语义记忆

| 字段 | 含义 |
|------|------|
| 知识条目数 | 向量数据库中存储的技术知识片段数量 |
| 事实条目数 | 向量数据库中存储的对话事实数量 |

### 技能

| 字段 | 含义 |
|------|------|
| 技能总数 | 助手已注册的技能数量 |
| 已验证数 | 经过验证、可信度较高的技能数量 |

---

## 近期请求记录（底部表格）

显示最近 20 条 API 请求的详细信息：

| 列名 | 含义 |
|------|------|
| **路径** | 请求的 API 地址（如 `/api/chat`） |
| **方法** | HTTP 方法（GET / POST / DELETE） |
| **状态码** | 响应状态（200 = 成功，4xx/5xx = 错误） |
| **耗时(ms)** | 该请求的处理时间，单位毫秒 |

状态码为 400 及以上时，该行会以**红色**高亮显示，便于快速定位问题。

### 常见状态码说明

| 状态码 | 含义 | 处理建议 |
|--------|------|---------|
| `200` | 请求成功 | 正常 |
| `422` | 请求参数格式错误 | 刷新页面重试 |
| `500` | 服务器内部错误 | 检查终端是否有报错信息 |

---

## 监控数据说明

- 监控数据**仅保存在内存中**，服务重启后清零，不影响对话记录
- 页面不会自动刷新，需手动刷新浏览器获取最新数据
- 记忆系统的数量统计反映的是**累计积累量**，数字越大说明助手对你的了解越深
